miniONNXRuntime · project snapshot

把 ONNX 推理拆成一条能看懂的最小主线

这个项目的目的很简单：从模型解析、图优化，到 CPU 执行、YOLO 后处理，再到内存生命周期分析， 每一步都能直接观察，而不是只看一个黑盒结果。
1 · Parse

LoadOnnxGraph

把 `.onnx` 转成内部 Graph，先把图结构、initializer、value_info 看清楚。

  • 图结构
  • 节点属性
  • 拓扑顺序
→
2 · Optimize

GraphOptimizer

做常量折叠、死节点清理和形状简化，让图更短、更容易执行。

  • ConstantFolding phase4
  • DeadNodeCleanup
  • ShapeSimplification
→
3 · Run

Session + Kernels

在 CPU 上执行节点，最后输出 detections 和可视化结果。

  • Session::Run
  • CPU Kernels
  • YOLO Postprocess

Memory story

initializer 先按需 materialize，再做 dead tensor eviction 和 buffer reuse。 这张图的重点是：让“图在怎么变”和“内存怎么变”都能直观看到。

Before: 权重和中间值常驻，峰值更高。
Now: runtime 只保留当前路径需要的少量张量。
Next: 再往 ExecutionProvider / Mac backend 走。
